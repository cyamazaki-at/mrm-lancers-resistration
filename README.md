# MRM フリーランサー登録・管理システム（Phase 1）

株式会社アイトリガー MRM事業向け。フリーランサー（ランサー）の登録環境と、社内エージェントの管理画面です。

## 起動方法

Node.js 22.5 以上（推奨: 24系）があれば **npm install 不要** でそのまま動きます。

```
cd mrm-app
node server.js
```

- ランサー向け画面: http://localhost:3000/
- 社内管理画面: http://localhost:3000/admin

ポートを変える場合は環境変数 `PORT` を指定してください。

## 実装済み機能（Phase 1）

### ランサー向け
- トップページ（サービス紹介 → 登録導線）
- 登録フォーム（入力 → 確認画面 → 完了。メール重複チェック・必須チェックあり）
- ログイン / ログアウト
- マイページ（登録内容とステータスの確認のみ。編集は Phase 3 予定）

### 社内エージェント向け（/admin）
- ログイン認証（エージェントのみアクセス可能）
- ランサー一覧: キーワード検索（氏名・メール・経歴）、スキル / ステータス / 稼働日数 / 担当者での絞り込み、並び替え
- ランサー詳細: 全項目表示、ステータス更新、担当エージェント割り当て、社内メモ、対応履歴（連絡ログ + ステータス変更等の自動記録）
- CSVエクスポート（絞り込み条件を引き継いで出力、Excel対応のBOM付きUTF-8）
- エージェント管理（管理者権限のみ）: アカウント一覧・新規追加

## 技術構成

- Node.js 標準モジュールのみ（外部依存なし）
  - DB: `node:sqlite`（SQLite / `data/mrm.db` に保存。バックアップはこのファイルのコピーでOK）
  - パスワード: `crypto.scrypt` によるソルト付きハッシュ
  - セッション: HttpOnly / SameSite=Lax Cookie + DBセッションテーブル
- テーブル構成（Phase 2 での案件テーブル追加を見越して正規化済み）
  - `lancers` / `skills` / `lancer_skills` / `agents` / `notes`（社内メモ）/ `activities`（対応履歴）/ `sessions`

## 本運用に向けた残課題（Phase 1.5 として推奨）

- HTTPS 化（社外公開する場合は必須。リバースプロキシ配下での運用を想定）
- CSRF トークンの導入（現状は SameSite Cookie による緩和のみ）
- パスワードリセット機能（現状はメール送信基盤がないため未実装）
- レート制限（ログイン試行回数の制限）
- `data/mrm.db` の定期バックアップ運用
